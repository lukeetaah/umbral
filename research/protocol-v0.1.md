# Protocol v0.1 — sound–color acquisition

**Status:** research draft; the local demo is a usability preview, not recruitment.
**Primary question:** Does a newly generated auditory cue produce above-sham, repeatable reporting of a paired color after the color is removed?

## Preflight

1. Read purpose, limits and existence of hidden controls.
2. Consent to an audible, nonclinical session with immediate withdrawal.
3. Set device volume below 60% and complete the 440-Hz audible check.
4. Stop if the tone is uncomfortable or cannot be heard at a comfortable setting.

## Sequence

Baseline → paired A1+C1 → hidden A1 repeat → sham → fading (65%, then 35%) → visual omission with A1 → transfer A2→A1 → A2-only test → delayed retest in a separately scheduled session.

The production protocol uses seeded constrained randomization while preserving acquisition before omission. The current Atlas demo contains two sham controls; it must not be analyzed as this preregistered protocol.

## Measures

Occurrence-first response, response class only afterward, confidence 0–100, pretrial expectation 0–100, optional short description under separate text consent, playback state, latency, interruption and adverse event.

## Safety and failure criteria

No advantage over sham, instability at retest, dependence on explicit expectation, excessive attrition, playback failures or a safety signal refute or invalidate the protocol as specified. No result licenses a claim of neural activation, real taste, memory creation or clinical benefit.
