# Capability audit — v0.1-alpha.2

Audited 2026-07-17 against the definitive UMBRAL directive. This file is deliberately conservative: “implemented” means a user can exercise the capability in the current repository; “scaffolded” means the architecture or documentation exists but is not a participant-ready study.

## Implemented and tested

- Next.js App Router, React, strict TypeScript, Tailwind/PostCSS, Zod, Vitest, Playwright, IndexedDB, Web Audio and AudioWorklet.
- Account-free local demo, explicit export and deletion, offline shell, no microphone/camera/uploads, no paid AI or audio API.
- Four differentiated local protocols: Atlas random mapping, Apprenticeship acquisition/fading, State Gate preparation/load, and Anchor safe-state progression.
- Audible preflight, conservative procedural synthesis, white/pink/brown noise, harmonics, filters, amplitude/frequency modulation, monaural/binaural beat modes, limiter, anti-click ramps, pause/resume internals, Stop button and Escape shortcut.
- Hidden sham, hidden repetitions, omission, transfer, expectation, confidence, optional short free text and honest single-session result language.
- Local deterministic Samuel roles with decision provenance and uncertainty.
- Public explanations, limits, glossary, privacy, consent, paper print view, living research files and evidence matrix.
- Compact Supabase schema, RLS, immutable published protocol versions, append-only algorithm provenance, append-only consent history with withdrawal, disabled chocolate flag.
- AGPL-3.0, governance/community/security files, fast CI and manual browser validation workflow.

## Scaffolded or partial

- Supabase sign-in and explicit private session-snapshot sync exist, but collective research recruitment, protocol publication UI, consent-version UI, normalized trial upload and researcher administration are not participant-ready.
- English exists for the home page, public explanation articles and demo flow. The canonical living paper and operational documentation remain primarily Spanish/English mixed and need a reviewed parallel translation before public recruitment.
- The local possibility engine is auditable but intentionally simple; regression, nearest-neighbor, bootstrap and bandit modules are research roadmap items rather than hidden claims.
- Offline behavior caches the shell after a successful first load; offline installability and update UX are not a full PWA product.
- RLS is statically tested. A real Supabase test project is required for end-to-end policy tests.

## Not implemented or intentionally disabled

- Public GitHub repository/release and Vercel deployment: blocked only by missing external authentication/setup; exact human actions are in `SETUP_REQUIRED.md`.
- Collective data collection, ethics approval, participant recruitment and scientific claims.
- Chocolate Pathway, food studies, audio uploads, recording, multimedia, neural measurements, clinical use and paid services.
- Absolute acoustic calibration, because browsers cannot measure ear-level dB SPL.

## Release blockers before real research

1. Independent scientific and ethics review of one narrow protocol.
2. Preregistration with primary outcome, exclusion rules, sample plan and stopping rule.
3. End-to-end RLS tests against a disposable Free project and verified deletion/withdrawal flows.
4. Human accessibility test including keyboard, screen reader, contrast and hearing-safety comprehension.
5. Reviewed English translation of paper, consent and safety text.
